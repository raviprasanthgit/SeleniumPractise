Patterns to solve theoretical problems:

- *3P Answers*
- *Technical* | *Selenium* | *Project*

For example, to explain interface concept in the interview - use these hints:

*Technical:*

- only contain method signatures and fields.
- cannot contain an implementation; so, there can be no constructor.
- cannot create object for interface

*Selenium:*

Some examples here:

- WebDriver, WebElement, Alert, TargetLocator

*Project:*

In our customized selenium automation framework, we implemented interface with:

- WdMethods (abstract methods for Selenium)
